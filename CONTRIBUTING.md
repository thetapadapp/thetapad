# Contributing to Thetapad

Thank you for your interest in contributing to Thetapad! This document provides guidelines for contributing feedback and ideas.

## Ways to contribute

### Reporting bugs

Good bug reports help us fix issues quickly. Please include:

- **Clear title** - Summarize the issue in a few words
- **Steps to reproduce** - Detailed steps to recreate the bug
- **Expected vs actual behavior** - What should happen vs what happens
- **Environment** - Browser, OS, and any relevant details
- **Screenshots** - If applicable, visual evidence helps a lot

### Suggesting features

We love hearing your ideas! When suggesting features:

- **Explain the problem** - What pain point does this solve?
- **Describe your solution** - How do you envision it working?
- **Consider alternatives** - Have you thought of other approaches?
- **Provide context** - Why is this important to you?

### Asking questions

Before asking:

1. Check the [documentation](https://thetapad.com/help)
2. Search existing issues for similar questions
3. If still stuck, open a Question issue!

## Issue labels

| Label | Description |
|-------|-------------|
| `bug` | Something isn't working |
| `enhancement` | New feature or improvement |
| `question` | Request for help or clarification |
| `triage` | Needs initial review |
| `confirmed` | Bug has been verified |
| `planned` | Feature is on the roadmap |

## Code of Conduct

- Be respectful and constructive
- Keep discussions on-topic
- Help others when you can
- Assume good intentions

## Response times

We review issues regularly, but please be patient. We're a small team and prioritize based on impact and complexity.

## Thank you!

Every bug report, feature request, and question helps make Thetapad better. We appreciate your contribution to the community!
