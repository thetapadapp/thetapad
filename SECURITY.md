# Security Policy

## Reporting a Vulnerability

We take security seriously at Thetapad. If you discover a security vulnerability, please report it responsibly.

### How to Report

**Do NOT open a public issue for security vulnerabilities.**

Instead, please email security concerns to: **security@thetapad.com**

Or use GitHub's private vulnerability reporting:
1. Go to the [Security tab](https://github.com/thetapadapp/thetapad/security)
2. Click "Report a vulnerability"
3. Fill out the form with details

### What to Include

Please provide as much information as possible:

- Type of vulnerability (e.g., XSS, CSRF, data exposure)
- Steps to reproduce the issue
- Potential impact of the vulnerability
- Any suggested fixes (optional)

### What to Expect

- **Acknowledgment**: We will acknowledge receipt within 48 hours
- **Updates**: We will keep you informed of our progress
- **Resolution**: We aim to resolve critical issues within 7 days
- **Credit**: With your permission, we will credit you in our security advisories

### Scope

This security policy applies to:

- The Thetapad web application (thetapad.com)
- Official Thetapad repositories

### Out of Scope

- Vulnerabilities in third-party dependencies (please report to the respective project)
- Social engineering attacks
- Physical security issues
- Denial of service attacks

## Security Features

Thetapad is designed with privacy and security in mind:

- **Local compilation**: Documents are compiled in your browser, never sent to servers
- **End-to-end encryption**: Shared documents use E2E encryption
- **No tracking**: We don't track your document contents
- **P2P collaboration**: Real-time editing without centralized servers

## Supported Versions

We provide security updates for the latest version of Thetapad. As a web application, users always have access to the most recent version.

Thank you for helping keep Thetapad secure!
